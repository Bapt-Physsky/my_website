1. Introduction

1.A. Introduction

Utiliser LaTeX pour la rédaction de document, notamment de rapport, de cours (voir notes de cours) ou bien encore produire un diaporama n'est pas si effrayant que ça. Cela demande juste de l'entraînement et de l'apprentissage, comme pour tout, vous me le ferais remarquer, et je vous répondrais : "Perspicace n'est-ce pas ?".

Quand on est étudiant ou étudiante en sciences durs (Maths, Physique, Chimie, Informatique, Biologie, Géologie...) à l'Université ou dans une grande école, on se rends compte à quel point, surtout quand on écrit un rapport avec des pages et des pages d'équations et/ou de tableaux et/ou d'images, que les logiciels tels que "Word" ou "Libre Office", bien que facile d'utilisation et, quelques part, intuitif pour l'utilisateur lambda, s'avère quelque peut frustrant. Surtout quand une image ne se trouve plus à l'endroit où elle devrait être et modifie toute la mise en page, ou bien un calcul qui s'allonge tellement que la zone d'équation ne tient pas sur une seule page.

Je ne dis par que LaTeX doit être le seul et unique éditeur de document que TOUT LE MONDE DOIT UTILISER mais que dans le cas d'une personne impliquée ou nerd, que se soit scolairement ou professionnellement, dans le milieu scientifique principalement, et bien LaTeX peut vous enlever une certaine épine du pied !

1.B. Ce cours n'est pas un cours

Je n'ai pas la prétention de faire un cours magistral ou un enseignement universitaire sur l'utilisation de LaTeX dans le cadre de vos études, je ne suis qu'un étudiant en physique qui a envie de partager, en français, tout ce que j'ai appris mais également mon expérience dans le monde merveilleux des rapports et productions écrites scientifiques.

D'ailleurs, le fait que LaTeX soit un langage de programmation, nous y reviendrons plus loin, fait qu'il offre une immense communauté sur internet, dont les forums en HTML sans CSS les plus obscurs ou une vidéo Youtube seront dans l'ombre d'un doute vos meilleurs amis ! Mais, je peux garantir que cette page sera sourcée et travaillée de sorte qu'elle soit fiable. D'ailleurs, j'en profite pour glisser maintenant ma source principale, qui a été quasiment ma porte d'entrée dans le code LaTeX, ce site, en anglais : LaTeX-tutorial Avant de plonger dans le vif du sujet, et d'écrire vos premiers documents avec LaTeX, revenons sur ce qui fait la force de ce dernier !

2. MikTeX

2.A. C'est quoi MikTeX ?

Pour que LaTeX fonctionne correctement sur votre machine, il faut y installer une librairie (comme n'importe quel autre langage). Avant d'élaborer sur la librairie MikTeX, je vais aborder le cas des machines GNU/Linux, en effet, vous n'avez pas besoin de MikTeX pour que LaTeX y fonctionne, juste quelques commandes dans le terminal.

2.B. Installation

  1. Allez sur le site : latex-project.org
  2. Descendez un peu la page jusqu'au niveau où vous trouverez les différents OS
  3. Sous Windows, cliquez sur le texte en bleu "MikTeX"
  4. Vous arriverez sur la page d'accueil de la distribution MikTeX
  5. En haut de cette page, cliquez sur "download"
  6. Vous aurez le choix entre les différents OS, choisissez dans ce cas présent Windows
  7. Cliquez sur download, et puis, cela se passe comme une installation de logiciel

3. TexMaker

3.A. Pourquoi TeXmaker ?

TeX a beau être un langage de programmation, il nous faut pour cela un IDE (Integrated Development Environment) ou en français EDI (Environnement de Développement Intégré). Mon choix s'est porté sur TeXmaker, pourquoi ?

Tout simplement car c'est la première option qui s'est porté à moi en proposant une interface en français (mais c'est pas le point important, mais je le dis quand même), un mode sombre (OUI !) mais surtout, une visionneuse PDF intégrée, permettant de suivre la mise en forme de son document à chaque compilation de code.

J'ai écris plus haut que le but du WYSIWYM est que l'auteur se focalise sur le contenu de son document/présentation, et là, je vous sort un logiciel proposant de voir son texte évoluer au fur et à mesure que l'on travaille sur le code.

Et bien, ce choix s'explique tout simplement : cela me rassurais, quand je commençais LaTeX pour ne pas être perdu.

TeXmaker est un logiciel libre cross-platform (Linux, Windows, MacOsX) d'édition LaTeX depuis 2003. Libre car il est distribué sous la licence GPL License. Il est disponible sur le site suivant : xm1math.net.

3.B. Installation

3.B.1 Installation sous GNU/Linux

Je commence par Linux, car je travaille et je code en LaTeX sur ma machine Linux, précisément, Linux Mint. Du coup, l'installation se fera non pas par le site internet, mais par le terminal !

  1. Ouvrir le terminal, avec la combinaison de touche : CTRL + ALT + T
  2. Entrez, après NOM@NOM-MACHINE :~$ la commande : SUDO APT-GET INSTALL TEXMAKER
  3. Suivre ce que le terminal vous dis
  4. Voilà, mais ne fermez pas tout de suite le terminal
  5. Entrez la commande suivante : SUDO APT-GET INSTALL TEXLIVE-LANG-FRENCH
  6. Voilà, c'est prêt ! - C'est puissant le terminal sur Linux :)
3.B.2 Installer sur Windows

Avant d'installer Texmaker, il nous faut d'abord installer une librairie LaTeX, chose déjà faite avec MikTeX !

4. Quelques rudiments de codes

4.A. Hello World !

Voilà, tout est prêt, on peut commencer. Comme l'indique le paragraphe, nous allons donc faire le premier document selon la tradition des programmeurs pour tout premier code dans un langage, le célébrissime "Hello World !".

Le code LaTeX pour "Hello World !" se présente comme suit : 


                \documentclass{article}

                \begin{equation}
                Hello World !
                \end{equation}

Lorsque que vous le compilé, vous obtiendrais un fichier PDF d'une page (format A4 ou Letter) où vous verrez : Hello World ! ; avec le numéro de la page en bas, ajouté automatiquement par la classe du document, ici article.

Voilà, vous venez de faire votre premier document en LaTeX, et à partir de cela, nous pouvons fixer un point essentiel à retenir : votre document finale (ce que vous obtenez après la compilation) est compris entre : 


                \begin{document}

                \end{document}

5. Les bases de la mise en pages sur LaTeX

5.A. Les classes de documents

LaTeX traite plusieurs classes de document. Au début, on peut être perdu et confus, mais je vais - pour toi public - expliquer leurs atouts et différences. Je précise cependant que nous utiliserons quasi-uniquement les classes book et article. Un chapitre sera dédié à beamer pour faire des présentation à la fin.

5.A.1. Classe Book (Livre) & Report (rapport)

Comme son nom l'indique, cette classe est prévu pour la production d'un livre, avec des spécificités associées, telles que :

  1. une page de titre suivit d'une page blanche ;
  2. il se divise en parties, chapitres, sections, sous-sections, sous-sous-sections, paragraphes et sous-paragraphes ;
  3. les parties et chapitres commenceront toujours sur une page impaire (dite "belle page") ;
Penchons nous sur la présentation d'un code pour un livre, car c'est la base la plus complète pour un document.

5.A.2. Classe Article

C'est celle que nous venons d'utiliser dans notre premier code. Comparée à la classe book, vous avons :

  1. le titre est sur la même page que le texte ;
  2. la division chapitre n'est pas utilisable

5.A.3. Classe Beamer

La classe beamer est le "PowerPoint" pour LaTeX. Pour faire un diaporama / présentation avec LaTeX, cette classe est le meilleur choix, car elle est la plus évoluée parmi les classes équivalentes. Avec une documentation extrêmement riche sur internet, beamer vient également avec deux sous-packages : pgf et xcolor, qui permettent, respectivement, d'inclure des images et de définir des couleurs dans le document.
Il y a un chapitre dédié pour produire des présentation plus loin dans ce "cours".

5.A.4. Classe Lettre

Cette classe, dérivée de la classe letter permet de réaliser une lettre selon les règles françaises de typographie. Pour vous illustrer ce que cette classe permet, je vous mets ici le code d'exemple disponible sur wikibooks, et que je vous invite à copier-coller dans TeXmaker pour voir le résultat. 


                \documentclass[12pt]{lettre}

                \usepackage[utf8]{inputenc}
                \usepackage[T1]{fontenc}
                \usepackage{lmodern}
                \usepackage{eurosym}
                \usepackage[french]{babel}
                \usepackage{numprint}

                \begin{document}

                \begin{letter}{Service des réclamations}
                \name{Jean Râleur}
                \address{Jean Râleur \\ 4, rue du Bac à sable \\ 80886 Sassone-le-Creux}
                \lieu{Sassone-le-Creux}
                \telephone{01 02 03 04 05}
                \email{jean.raleur@fai.fr}
                \nofax

                \def\concname{Objet :~} % On définit ici la commande 'objet'
                \conc{rétractation}
                \opening{Madame, Monsieur,}

                Vous m'avez démarché la semaine dernière pour me proposer l'édition de luxe de l'encyclopédie Wikipédia, pour la somme de \numprint{5000}~\euro.
                Conformément à la loi m'accordant un délai de rétractation de 7~jours, je renonce à mon achat et demande le remboursement de la somme versée.

                \closing{Je vous prie d'agréer, Madame, Monsieur, mes salutations distinguées.}

                \end{letter}

                \end{document}
